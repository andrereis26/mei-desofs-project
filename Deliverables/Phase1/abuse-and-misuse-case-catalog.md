# Phase 1 - Abuse and Misuse Case Catalog

## 1. Abuse and Misuse Case Catalog

| ID | Type | Case | Threat Actor | Related Threat ID(s) | Explanation | Planned Mitigation Direction | Planned Verification |
|---|---|---|---|---|---|---|---|
| AB-001 | Abuse | Credential stuffing on login endpoints | External attacker | T-001, T-011 | The attacker replays leaked credential pairs at scale; successful matches cause account takeover and unauthorized data access. | Anti-automation controls, lockout/rate-limit policy | Auth abuse test scenarios in security test plan |
| AB-002 | Abuse | Automated mass registration | External attacker | T-003, T-011 | Bot-driven account creation consumes resources and can prepare spam or fraud campaigns, degrading platform trust and availability. | Registration throttling and abuse detection | Register abuse tests and monitoring requirements |
| AB-003 | Abuse | Token tampering/forgery attempts | External attacker | T-002, T-001 | The attacker alters or forges tokens to impersonate users; success bypasses authentication and authorization controls. | Strong signing secret and token signature validation | Negative token-validation tests |
| AB-004 | Abuse | Reuse of stolen token until expiration | External attacker | T-001, T-019 | A captured valid token is reused before expiry, enabling ongoing unauthorized API access without knowing user credentials. | Token lifecycle controls and revocation strategy | Revocation and expiry behavior tests |
| AB-005 | Misuse | Unauthorized profile access (IDOR) | Authenticated user | T-006, T-007 | A logged-in user manipulates resource identifiers to read another user's profile, violating confidentiality boundaries. | Ownership checks plus administrative exception policy | Role/ownership negative-path tests |
| AB-006 | Misuse | Unauthorized profile updates | Authenticated user | T-007 | A logged-in user targets update flows for profiles they do not own, enabling data tampering and possible privilege escalation. | Ownership checks and immutable privilege fields | Update authorization negative-path tests |
| AB-007 | Misuse | Department governance bypass | Authenticated manager/user | T-007, T-005 | A user invokes governance actions without required departmental authority, altering structure or memberships illegitimately. | Department-manager and admin authorization rules | Department authorization integration tests |
| AB-008 | Misuse | Unauthorized department document upload | Authenticated user | T-007, T-009, T-001, T-017 | A user uploads content into departments where they lack membership, bypassing governance controls and contaminating shared data. | Membership-validated upload policy | Upload authorization matrix tests |
| AB-009 | Abuse | Path traversal via filename/path inputs | Authenticated user | T-017 | Malicious path payloads (for example traversal sequences) attempt storage-root escape to read or overwrite unintended server files. | Canonical path verification and filename sanitization | Filesystem traversal attack tests |
| AB-010 | Abuse | Malicious content upload | Authenticated user | T-016 | The user uploads malware or weaponized files that can compromise systems when scanned, processed, or downloaded by others. | File validation and content security policy | Malicious upload detection tests |
| AB-011 | Misuse | Missing audit coverage for sensitive actions | Internal contributor | T-010 | Security-relevant code changes omit audit events, reducing detection capability and impairing forensic reconstruction. | Mandatory audit-control checklist in code review | Audit completeness unit/integration tests |
| AB-012 | Misuse | Unsafe initial privileged account provisioning | Insider/operator | T-004, T-002 | Weak bootstrap procedures create default or poorly controlled admin access, enabling persistent unauthorized privilege. | Controlled bootstrap and secret rotation policy | Environment readiness/security checklist |
| AB-013 | Abuse | SQL injection via query/body parameter manipulation | External attacker or authenticated user | T-005, T-006 | The attacker sends crafted input containing SQL operators or clauses to change query behavior, which can leak, modify, or delete protected records. | Parameterized queries, ORM-safe data access patterns, and strict validation of queryable inputs | SQL injection negative-payload tests plus repository/query-construction review |
| AB-014 | Abuse | Supply-chain exploitation via vulnerable container image | External attacker or malicious contributor | T-009, T-013 | A vulnerable base image or dependency is introduced and later exploited to access data or disrupt service. | Container image scanning and registry release policy | Container image scan gate in CI |
| AB-015 | Misuse | Secrets exposure through CI/CD or repository storage | Internal contributor or attacker with repo access | T-001, T-002, T-009 | Secrets are stored in source control or CI variables, enabling token theft or privilege escalation. | Doppler-managed secrets and least-privilege access | Doppler integration checks and secret scanning |
